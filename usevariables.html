ARITHMETIC OPERATIONS WITH TEXT-TYPE VARIABLES

A sum of text strings results in a concatenation of these.

When we add numbers enclosed within a string, the result is a concatenation of the numbers and not an arithmetic sum.

When we multiply numbers enclosed within strings, javascript transforms the texts into numbers and correctly processes the multiplication.

This is precisely a very interesting feature of JS. Variables evolve in real time from one type to another. Unlike other languages.

ARITHMETIC OPERATIONS WITH VARIABLES OF TYPE NUMBER

The basic operations that the numbers admit are:

  • Sum, +
  • Subtraction, -
  • Division, /
  • Multiplication, *
  • module operation (the rest of the division), %
There are also expressions that are actually abbreviations of other longer expressions.

  • x++ equals x = x + 1;
  • ++x equals x = x + 1;
  • x– equals x = x – 1;
  • –x equals x = x – 1;
In either case it is essential to note that the sign of (=) refers to an assignment and not to an equality. There is no first degree equation. It is the form with which we indicate that a variable happens to have a new value resulting from operating with the one it had until now.

OPERATIONS WITH BOOLEAN TYPE VARIABLES

The basic operations that Booleans admit are:

  • Equality in content, ==
  • Equality in type of variable and content, ===
  • Negation, !
  • «Not the same as«, !=
  • And, &&
  • Or, ||
  • >= , greater than or equal; <, less than
  • <= , less than or equal; >, greater than
